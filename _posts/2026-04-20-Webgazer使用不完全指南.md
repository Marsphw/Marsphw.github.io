Webgazer是一个用于实时追踪被试眼动的jsPsych插件，通过计算机视觉技术对被试的眼睛以及其他面部特征进行识别，并预测注视点。该系统通过被试点击或注视屏幕上特定位置的方式进行校准，并利用回归（Regression）进行预测。
利用Webgazer收集眼动数据进行实验一般包括以下步骤：
1. 初始化摄像头
2. 校准（calibration）
3. 验证（validation）

## 基本使用

### 加载插件

**使用CDN托管的脚本**
```html
<head>
    <script src="https://unpkg.com/jspsych@latest"></script>
    <script src="https://unpkg.com/@jspsych/plugin-webgazer-init-camera@latest"></script>
    <script src="https://unpkg.com/@jspsych/plugin-webgazer-calibrate@latest"></script>
    <script src="https://unpkg.com/@jspsych/plugin-webgazer-validate@latest"></script>
    <script src="https://cdn.jsdelivr.net/gh/jspsych/jspsych@latest/examples/js/webgazer/webgazer.js"></script>
    <script src="https://unpkg.com/@jspsych/extension-webgazer@latest"></script>
</head>
```

**下载jsPsych并手动添加到项目中**
假设在下载之后你将`jspsych/dist`文件夹中的所有文件复制到了你的项目下的`jspsych`文件夹中，并将`jspsych/js/webgazer/webgazer.js`文件复制到了你的项目下的`webgazer`文件夹下。
```html
<head>
    <script src = "jspsych/jspsych.js"></script>
    <script src = "jspsych/plugin-webgazer-calibrate.js"></script>
    <script src = "jspsych/plugin-webgazer-init-camera.js"></script>
    <script src = "jspsych/plugin-webgazer-validate.js"></script>
    <script src = "webgazer/webgazer.js"></script>
    <script src = "jspsych/extension-webgazer.js"></script>
</head>
```

以上两种方式均可正常使用，依个人喜好即可。

### 初始化摄像头（init-camera）

我们可使用`webgazer-init-camera`插件来初始化摄像头，并帮助被试调整面部的摆放位置。在初始化摄像头时，页面上会呈现出摄像头所收集到的实时图片，并要求被试通过调整将面部摆放在指定的方框中，待方框变绿后方可进行下一步操作。
```javascript
let init_camera = {
    type: jsPsychWebgazerInitCamera,
    instructions: `
        <p>请确认摄像头是否已开启。</p>
        <p>请移动您的头部，使其位于屏幕中央的方框内。</p>
        <p>待方框变为绿色后，请点击继续按钮。</p>
    `,                        // 可自行修改
    button_text: "继续实验"    // 可自行修改
}
```

### 校准（calibration）

我们可使用`webgazer-calibration`插件对Webgazer进行校准，其分为两个模式，分别为
- click模式：利用被试的鼠标点击的坐标对眼动模块进行校正；
- view模式：利用所出现的点/图片所在位置对眼动模块进行校正。

Webgazer把获取的坐标作为用户确实在看的坐标点，把此刻从摄像头里提取到的眼部特征当作输入，用机器学习算法在线更新映射函数 $f: X \to y$ 。每获得一个坐标，就把 $(X, y)$ 当作训练样本，增量更新模型权重。随着获得的坐标点的数量的增加，映射函数逐渐收敛，预测误差降低。

官方内置了三种机器学习算法，即普通的岭回归（ridge）、加权岭回归（weightedRidge）和线岭回归（threadedRidge），可通过以下代码进行更换。
```javascript
// 在初始化Webgazer之后进行更换
webgazer.setRegression("ridge");
webgazer.setRegression("weightedRidge");
webgazer.setRegression("threadedRidge");
```

若想采用自己编写的机器学习算法，可自行动态注册新模块。
```javascript
// 构造函数
function MyRegressionor() {
    this.sample = [];
}

// addData函数
MyRegressionor.prototype.addData = function(eyeFeatures, screenPos) {
    this.sample.push({x: eyeFeatures, y: screenPos});
};

// 算法主体
MyRegressionor.prototype.predict = function(eyeFeatures) {
    // 各种机器学习算法均可使用，如kNN、神经网络等
    // 返回{x, y}即可
    return {x, y};
}

// 注册并切换
webgazer.addRegressionModule('myReg', MyRegressor);
webgazer.setRegression('myReg');
```

**插件中有以下参数可供设置：**
- `calibration_points`: 校正点的坐标，默认为(10%, 10%), (10%, 50%), (10%, 90%), (50%, 10%), (50%, 50%), (50%, 90%), (90%, 10%), (90%, 50%), (90%, 90%)。（此处的10%等意为点的位置位于屏幕的相对位置）
- `calibration_mode`: 可在click和view中选择一个，默认为click。
- `point_size`: 校正点的大小，默认为20px。
- `repetitions_per_point`: 每个点出现的次数，默认为1次。
- `randomize_calibration_order`: 校正点出现顺序是否随机，默认为不随机。
- `time_to_saccade`: 若使用view模式，则该参数为从校正点出现到开始校正的时间，默认为1000ms。
- `time_per_point`: 若使用view模式，则该参数为每个校正点用于校正的时间，默认为1000ms。

该插件中默认的校正点的颜色为黑色，如需要对校正点的颜色进行更改，可对`plugin-webgazer-calibrate.js`中的`calibration_display_gaze_only`函数进行修改。修改函数第一行中的`border`和`background-color`即可。

若需要使用外部图片，可参照以下代码对`calibration_display_gaze_only`函数进行修改。

```javascript
const calibration_display_gaze_only = (pt) => {
   const pt_html = `
      <img id="calibration-point"
         src="../img/blue.png"
         style="width:${trial.point_size}px;
                  height:${trial.point_size}px;
                  position:absolute;
                  left:${pt[0]}%;
                  top:${pt[1]}%;">
   `;
// 之后的代码保持相同即可
};
```

### 验证（validation）

我们可使用`webgazer-validate`插件对Webgazer在经过校正之后的准确率进行验证，形式与校正阶段基本类似。

插件中有以下参数可供设置：
- `validation_points`: 验证点的位置，默认值与calibration相同。
- `validation_point_coordinates`: 可从percent和center-offset-pixels当中进行选择，前者表示点的位置指定为屏幕百分比，后者则表示点的位置指定为距离屏幕中心的距离。
- `roi_radius`: 验证为预测正确的范围，单位为px，默认值为200，即若预测点和验证点之间的距离不超过200px则记为预测正确。
- `randomize_validation_order`: 意义和默认值与calibration中所述相同。
- `time_to_saccade`: 意义和默认值与calibration中所述相同。
- `validation_duration`: 意义与calibration中所述相同，默认值为2000ms。
- `point_size`: 意义和默认值与calibration中所述相同。
- `show_validation_data`: 是否在验证阶段结束之后展示数据，默认为不展示。

该插件中默认的验证点同样为黑色，若需对颜色进行修改，可修改`plugin-webgazer-validate.js`中的`drawValidationPoint_PercentMode`与`drawValidationPoint_CenterOffsetMode`函数，修改方式与calibration中类似。

## 后记

Webgazer是一个强大的心理学实验中可使用的眼动探测库，若想直接使用的话上手并不难，只需要一些简单的javascript与jsPsych知识。但若需要对实验的眼动模块进行个性化设计的话则需要对源代码进行一定程度上的修改，即对部分函数进行修改并添加部分参数。
在jsPsych官方文档概述中的眼动部分最后，给出了一个可以直接使用的示例，并同时展示了如何根据验证阶段的数据判断是否需要重新校准。在jsPsych的`/example`文件夹中，也有`webgazer.html`, `webgazer_image.html`, 和`webgazer_audio.html`作为示例可供直接运行。
